# Package: google/wire — Complete Reference

## Import
```go
// wire.go files only — uses build tag to exclude from normal builds
//go:build wireinject

import "github.com/google/wire"
```

## 1. Provider Functions

```go
// A provider is any function that constructs a value.
// Signature: func(...deps) (T, error) OR func(...deps) (T, func(), error)
// The func() cleanup is called on shutdown.

// Simple provider — no cleanup needed
func NewOrderRepository(pool *pgxpool.Pool) *postgres.OrderRepository {
    return postgres.NewOrderRepository(pool)
}

// Provider with cleanup
func NewPool(ctx context.Context, cfg *config.Config) (*pgxpool.Pool, func(), error) {
    pool, err := pgxpool.New(ctx, cfg.Database.DSN)
    if err != nil { return nil, nil, fmt.Errorf("NewPool: %w", err) }
    cleanup := func() { pool.Close() }
    return pool, cleanup, nil
}

// Provider that binds an interface
// (see Interface Binding section below)
func NewKafkaWriter(cfg *config.Config) (*kafka.Writer, func(), error) {
    w := kafka.NewWriter(kafka.WriterConfig{Brokers: cfg.Kafka.Brokers})
    return w, func() { w.Close() }, nil
}
```

## 2. Provider Sets

```go
// Group related providers into sets for reuse
// infrastructure/wire.go (NO build tag — this is exported, not the injector)

package infrastructure

import "github.com/google/wire"

var DBProviderSet = wire.NewSet(
    NewPool,
    NewOrderRepository,
    NewUserRepository,
)

var KafkaProviderSet = wire.NewSet(
    NewKafkaWriter,
    NewKafkaReader,
    NewEventPublisher,
)

var RedisProviderSet = wire.NewSet(
    NewRedisClient,
    NewUserCache,
    NewSessionStore,
)

var InfrastructureSet = wire.NewSet(
    DBProviderSet,
    KafkaProviderSet,
    RedisProviderSet,
)
```

## 3. Injector Functions (wire.go)

```go
// cmd/api/wire.go — build tag prevents compilation in normal builds
//go:build wireinject

package main

import (
    "github.com/google/wire"
    "github.com/org/project/internal/adapters/http"
    "github.com/org/project/internal/app/order"
    "github.com/org/project/internal/infrastructure"
)

// InitializeApp is the injector — wire fills in the implementation
func InitializeApp(ctx context.Context, cfg *config.Config) (*App, func(), error) {
    wire.Build(
        // Infrastructure
        infrastructure.InfrastructureSet,

        // Application layer
        order.NewCreateOrderHandler,
        order.NewGetOrderHandler,
        order.NewCancelOrderHandler,

        // HTTP adapters
        http.NewOrderHandler,
        http.NewUserHandler,
        http.NewRouter,

        // App wiring
        NewApp,
    )
    return nil, nil, nil  // wire replaces this body
}
```

## 4. Interface Binding

```go
// When a provider returns a concrete type but the consumer needs an interface:
// wire.Bind(new(InterfaceType), new(*ConcreteType))

var RepositorySet = wire.NewSet(
    postgres.NewOrderRepository,
    // Bind concrete *postgres.OrderRepository to domain.order.Repository interface
    wire.Bind(new(order.Repository), new(*postgres.OrderRepository)),

    postgres.NewUserRepository,
    wire.Bind(new(user.Repository), new(*postgres.UserRepository)),
)

// Alternative: provider returns the interface directly (preferred when possible)
func NewOrderRepository(pool *pgxpool.Pool) order.Repository {
    return postgres.NewOrderRepository(pool)  // returns interface
}
```

## 5. Value Providers

```go
// Inject a constant value that wire can't construct
wire.Build(
    // Inject a static string value
    wire.Value("v1.0.0"),

    // Inject from interface (value not constructed — taken as-is)
    wire.InterfaceValue(new(io.Writer), os.Stdout),
)
```

## 7. Testing with Wire

```go
// test/wire_test.go — separate injector for tests (uses mocks)
//go:build wireinject

package integration_test

func InitializeTestApp(ctx context.Context) (*App, func(), error) {
    wire.Build(
        // Real infrastructure (testcontainers)
        testutil.NewTestPool,     // spins up postgres container
        testutil.NewTestRedis,    // spins up redis container

        // Real application layer (not mocked in integration tests)
        order.NewCreateOrderHandler,

        // Mock external services
        wire.Bind(new(app.Notifier), new(*mocks.MockNotifier)),
        mocks.NewMockNotifier,

        NewTestApp,
    )
    return nil, nil, nil
}
```

## Wire Workflow

```bash
# Install
go install github.com/google/wire/cmd/wire@latest

# Generate (run from project root)
wire ./cmd/api/
wire ./cmd/worker/

# Add to Makefile
generate:
    wire ./...
    go generate ./...

# wire is a check-in tool — commit wire_gen.go
# wire_gen.go is the source of truth for DI graph
```

## Wire Checklist
- [ ] `//go:build wireinject` on all wire.go files
- [ ] `wire_gen.go` committed to version control
- [ ] Provider sets group related providers (`DBSet`, `KafkaSet`, etc.)
- [ ] Cleanup functions returned for all resources (pool, connections, writers)
- [ ] `wire.Bind` used when provider returns concrete but consumer needs interface
- [ ] `wire ./...` run in CI to verify no missing providers
- [ ] Test injectors use testcontainers for real infrastructure
- [ ] No side effects in provider functions — just construction
- [ ] All providers accept `context.Context` as first arg when doing I/O on startup
